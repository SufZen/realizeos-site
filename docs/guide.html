1. Test Everything Works

Run through these tests right now to confirm your system is properly configured. It's much easier to catch and fix issues early than after you've built on top of the system.

Test 1: Brand Recognition

Ask your system:

What do you know about my business?

Expected: The response references your brand identity, business name, audience, and values — not generic AI filler. If it gives generic output, your brand-identity.md needs more detail.

Test 2: Voice Calibration

Ask your system:

Write a social media post about why AI agents should work as a team, not as disconnected tools.

Expected: The output matches your voice rules. If you said "no exclamation marks," there should be none. If you said "short paragraphs," they should be short. If it sounds generic, your brand-voice.md needs more specific rules.

Test 3: System Health Full edition only

Open in your browser:

http://localhost:8080/status

Expected: A JSON response showing "healthy" status, connected LLM providers, and active systems.

Test 4: Agent Routing Full edition only

curl -X POST http://localhost:8080/api/chat \
  -H "Content-Type: application/json" \
  -d '{"message": "Analyze the competitive landscape for [your industry]", "system_key": "consulting", "user_id": "test-user"}'

Expected: The response is handled by the Analyst agent (check the agent field), using a more capable model for complex analysis.

All tests pass?

You're in good shape. Move to First Steps to start building. If something's off, jump to Troubleshooting.

2. First Steps After Setup

Your system is running. Here's a roadmap for making the most of your first week — organized by what has the highest impact at each stage.

Day 1 — Foundations

  • Send 5-10 test messages to check your brand voice — refine the rules based on what you see
  • Complete your brand identity file (F-foundations/brand-identity.md) — add specifics you might have skimped during setup
  • Write 5-10 concrete voice rules in F-foundations/brand-voice.md — the more specific, the better
  • Key tip: Add 3-5 anti-patterns (what your AI should NEVER say or do). These have outsized impact.

Day 2-3 — Knowledge

  • Add domain knowledge to B-brain/ — industry data, methodologies, your past work, client context
  • Test that agents reference this knowledge in their responses
  • Use the ingestion command to import existing documents you already have

Day 4-5 — Workflows

  • Create your first YAML skill — a content pipeline (writer → reviewer → save) is a great starting point
  • Test it with a real task from your actual workflow
  • Iterate on agent prompts based on output quality — small prompt changes make big differences

Week 2+ — Expansion

  • Connect Google Workspace tools (email, calendar, drive)
  • Add specialized agents for roles you need (researcher, ops, client comms)
  • Build skill workflows for your top recurring tasks
  • Let the self-evolution engine suggest improvements based on your usage patterns

What's coming to your inbox

  • Day 3: Tips for writing great brand voice rules — the patterns that actually work
  • Day 7: Advanced workflows + connecting Google Workspace
  • Day 14: We'll check in — how's it going?

4. Troubleshooting

Common issues and their fixes. If your problem isn't here, ask in the community or email info@realizeos.ai.

Installation Issues

"git is not recognized" or "git: command not found"

Git isn't installed. Download from git-scm.com. After installing, close and reopen your terminal for the PATH to update.

"python: command not found" or wrong Python version

Try python3 --version instead. If Python isn't installed, download from python.org. During installation, check "Add Python to PATH." This is the most commonly missed step.

"pip: command not found"

Use python -m pip install -r requirements.txt instead. If that doesn't work, reinstall Python with the "pip" option checked.

Docker Desktop won't start

Windows: Make sure WSL2 is installed. Open PowerShell as admin and run wsl --install. Restart your computer.
Mac: Make sure you downloaded the right version (Apple Silicon vs Intel).
Both: Check that virtualization is enabled in your BIOS settings. Make sure Docker Desktop is actually running (check the system tray icon).

"Port 8080 already in use"

Another process is using that port. Either stop it, or change the port in your .env file: set API_PORT=8081.

"Obsidian shows empty vault"

You opened the wrong folder. Close the vault, then re-open by selecting the folder that contains the FABRIC subfolders (A-agents/, B-brain/, etc.). If you unzipped into a nested folder, look one level deeper.

"claude: command not found"

Node.js isn't installed or isn't in your PATH. Close and reopen your terminal after installing Node.js. On macOS: brew install node. On Windows: download from nodejs.org and check "Add to PATH" during install.

API Key Issues

"Invalid API key" or "Authentication failed"

Check your .env file. Common mistakes:

  • Spaces around the = sign (wrong: ANTHROPIC_API_KEY = sk-...)
  • Quotes around the value (wrong: ANTHROPIC_API_KEY="sk-...")
  • Trailing spaces or invisible characters — re-type the key manually
  • Key was revoked — create a new one at console.anthropic.com

"Insufficient credits" or "Payment required"

Your Anthropic account needs credit. Go to console.anthropic.com → Billing → Add at least $5. Usage is pay-as-you-go, typically $2-8/month.

"Rate limit exceeded"

You're sending too many requests too quickly. Wait a minute and try again. For production use, adjust RATE_LIMIT_PER_MINUTE in your .env or increase your tier limits in the Anthropic console.

Runtime Issues

"Output sounds generic — not my voice"

Your F-foundations/brand-voice.md is too vague. Vague instructions like "be professional" don't help. Write specific rules:

  • "Use short paragraphs (2-3 sentences)"
  • "Start with data or a specific observation, not a generic claim"
  • "No exclamation marks"
  • "Never use the word 'synergy'"
  • Also paste a real writing sample from your own work into the "Good Example" section

"Agents respond generically — don't reference my knowledge base"

The knowledge base might not be indexed. Run:

python cli.py index

Also check that your agent definitions in A-agents/ have specific, detailed role descriptions — not just "handles content."

"Skill doesn't trigger"

Check the YAML syntax in your skill file. Common issues: incorrect indentation, missing triggers array, or the trigger keywords don't match your message. Test with an exact trigger phrase first. YAML is sensitive to spacing — use a validator if unsure.

"KB search returns nothing"

Run python cli.py index to rebuild the index. Also check that your knowledge files use UTF-8 encoding and have actual content — not just headers and empty sections.

"Claude Code doesn't read CLAUDE.md"

Make sure you're running claude from the correct directory — the one containing the CLAUDE.md file. Check with ls (Mac/Linux) or dir (Windows) that the file exists in your current working directory.

"Permission denied" (macOS)

macOS may block terminal access to your files. Go to System Settings → Privacy & Security → Full Disk Access → Enable your terminal app (Terminal, iTerm2, etc.).
