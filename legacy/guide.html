1. Before You Start — What to Prepare

Gather these before you begin. Having them ready means you can go from download to a working AI team in one sitting.

Preparation Checklist

  • Your business name(s) — If you run multiple ventures, list each one. RealizeOS creates isolated systems per venture.
  • Brand voice sample — Any existing content that sounds like "you." A LinkedIn post, email, proposal — anything. This is the single most impactful thing for quality.
  • 2-3 workflows to automate first — What do you repeat weekly? Content creation? Client proposals? Meeting prep? Be specific.
  • API keys (Full edition only) — Anthropic API key ($5+ credit) and/or Google AI API key (free tier). See Section 4.
  • A computer you control — No corporate restrictions needed. RealizeOS runs locally. No data leaves your machine except LLM API calls.

2. Recommended: Brand Discovery Worksheet

~10 minutes

Before diving into setup, take 10 minutes to complete the Brand Discovery Worksheet. It asks 10 questions about your business, audience, and voice — and outputs a markdown file you can paste directly into your vault.

This makes Step 3 (defining your brand) much faster. You'll already have the answers ready to paste.

Why this matters

The #1 reason people get generic AI outputs: they skip the brand definition step. 10 minutes here saves hours of frustration later. Open the worksheet →

3. Lite Edition Setup

~15 minutes total

The Lite edition runs entirely through Obsidian and Claude Code. No servers, no Docker, no coding. Your knowledge base lives as markdown files that Claude reads and works from.

Prerequisites

  • Obsidian (free) — Download from obsidian.md
  • Claude Pro subscription ($20/mo) — From claude.ai. This gives you Claude Code access.

Important: Save location

Save the vault in your Documents folder, not in a cloud-synced folder (OneDrive, iCloud, Google Drive). Cloud sync can cause conflicts with Obsidian. You can back it up manually or with git later.

Step 1: Download and Open 2 min

  1. Download the RealizeOS Lite package from your Google Drive link (received after purchase).
  2. Unzip the file to your Documents folder.
  3. Open Obsidian → Click "Open folder as vault" → Select the realize_lite/ folder.

What you should see

  • Obsidian opens with a sidebar showing the FABRIC folder structure
  • You can see folders: A-agents/, B-brain/, C-creations/, F-foundations/, I-insights/, R-routines/
  • A CLAUDE.md file exists at the root
  • If you see an empty vault → you opened the wrong folder. Make sure you selected the folder that contains these subfolders.

Step 2: Define Your Identity 3 min

  1. Open shared/identity.md in Obsidian
  2. Fill in your name, role, values, and communication preferences
  3. This tells the AI who you are and how you like to work

What you should see

  • A markdown file with clear sections to fill in
  • Guided prompts telling you what to write in each section
  • If you completed the Brand Worksheet, paste the relevant sections here

Step 3: Define Your Brand 5 min

  1. Open F-foundations/brand-identity.md — Fill in business name, mission, audience, values
  2. Open F-foundations/brand-voice.md — Define how your content should sound (tone, vocabulary, dos and don'ts)

Tip: Paste an example of writing that nails your voice into the "Good Example" section. This is the most impactful thing you can do for output quality.

What you should see

  • Two markdown files with structured templates
  • Clear guidance on what to write in each section
  • If you did the Brand Worksheet, most of this is copy-paste from your exported markdown

Step 4: Review Your Agents 2 min

  1. Browse A-agents/ — you'll find four pre-built agents
  2. Orchestrator — General coordinator (good default)
  3. Writer — Content creation
  4. Reviewer — Quality gatekeeper
  5. Analyst — Research and strategy

Each agent is just a markdown file. Read through them — customize the role descriptions if you want, or leave the defaults for now.

Step 5: Launch Claude Code and Test 3 min

  1. Open your terminal in the realize_lite/ directory
  2. Run Claude Code — it automatically reads your CLAUDE.md file
  3. Try this test prompt:
Write a LinkedIn post about the importance of building systems, not just using tools.

What you should see

  • Claude identifies itself as your Orchestrator agent (or routes to the Writer)
  • The output is written in your brand voice (not generic ChatGPT style)
  • Content references your knowledge base files
  • The Writer drafts, then the Reviewer checks quality
  • If the output sounds generic → your brand-voice.md is probably too vague. Add specific rules like "Use short paragraphs. Start with data, not opinions."

That's it. Your Lite edition is running. Go to First Steps After Setup for your week-1 roadmap.

4. Full Edition Setup

~30 minutes total

The Full edition gives you the complete engine: multi-LLM routing, API server, tool integrations, and self-evolution. Most of this setup is copy-paste commands.

Prerequisites

  • Python 3.11+ — Check with python --version
  • Docker (recommended) — docker.com
  • Git — Check with git --version
  • API keys — Anthropic (required) + Google AI (recommended). See below.

Don't have Python or Git?

Python: Download from python.org. During installation, check "Add Python to PATH."
Git: Download from git-scm.com. Default options are fine.
Docker: Download Docker Desktop from docker.com. Make sure it's running before you continue.

Getting Your API Keys 5 min

Lite edition uses your Claude Pro subscription — no API keys needed. Full edition requires these:

Anthropic API Key (Claude models)

  1. Go to console.anthropic.com
  2. Create an account or sign in
  3. Navigate to API Keys and create a new key
  4. Add $5-10 credit (pay-as-you-go, no subscription required)
  5. Save the key somewhere safe — you'll need it in a few minutes

Typical cost: $2-8/month depending on usage. Claude Sonnet handles most tasks.

Google AI API Key (Gemini Flash — optional but recommended)

  1. Go to aistudio.google.com/apikey
  2. Sign in with your Google account
  3. Click "Create API Key"
  4. Copy the key — done

Gemini Flash has a generous free tier. It handles quick tasks so Claude can focus on complex work.

Step 1: Clone the Repository 2 min

Option A: Docker (recommended)

git clone https://github.com/SufZen/realize-os.git
cd realize-os

What you should see

  • Terminal shows "Cloning into 'realize-os'..." followed by download progress
  • A new folder called realize-os/ appears in your current directory
  • If you see "git is not recognized" → Install Git first (see Prerequisites above)
  • If you see "Permission denied" → Try without sudo, or check your SSH keys

Step 2: Configure Your Environment 5 min

cp .env.example .env

Now open .env in any text editor and add your API keys:

ANTHROPIC_API_KEY=sk-ant-your-key-here
GOOGLE_AI_API_KEY=your-google-key-here

What you should see

  • A .env file in your project root with your keys filled in
  • No spaces around the = sign
  • No quotes around the key values
  • If the file is empty → you may have run cp from the wrong directory. Run ls to check you're in the realize-os/ folder.

Step 3: Initialize from a Template 2 min

python cli.py init --template consulting

Other templates available: agency, multi-venture, saas, ecommerce. Pick the one closest to your business.

What you should see

  • Terminal shows "Initializing system from template: consulting..."
  • FABRIC directories are created under your system folder
  • Template files are populated with starter content
  • If "python not found" → try python3 cli.py init --template consulting

Step 4: Start the Server 2 min

With Docker (recommended):

docker compose up

Without Docker:

pip install -r requirements.txt
python cli.py serve

What you should see

  • Docker: containers building and starting, logs flowing
  • Local: dependencies installing, then "Server running at http://localhost:8080"
  • No red error messages
  • If Docker won't start → Make sure Docker Desktop is running. Try docker compose down then docker compose up --build
  • If "port 8080 in use" → Change the port in your .env file or stop the other process using that port

Step 5: Send a Test Message 3 min

curl -X POST http://localhost:8080/api/chat \
  -H "Content-Type: application/json" \
  -d '{
    "message": "Help me plan a content strategy for next quarter",
    "system_key": "consulting",
    "user_id": "test-user"
  }'

What you should see

  • A JSON response with an AI-generated strategy
  • The response includes an agent field showing which agent handled it
  • Content matches the brand voice defined in your system's Foundations
  • If "connection refused" → Server isn't running. Check the terminal where you ran docker compose up for errors.
  • If "API key error" → Check your .env file for typos. No spaces around =.

Step 6: Customize Your Brand 10 min

  1. If you completed the Brand Worksheet, paste your exported markdown into the relevant FABRIC files
  2. Edit F-foundations/brand-identity.md — your business description, audience, values
  3. Edit F-foundations/brand-voice.md — tone, vocabulary, examples, anti-patterns
  4. Review agent definitions in A-agents/ — customize roles for your needs
  5. Restart the server to pick up changes

Your Full edition is running. Go to First Steps After Setup for your week-1 roadmap.

6. First Steps After Setup

Your system is running. Here's a roadmap for making the most of your first week.

Day 1 — Foundations

  • Complete your brand identity file (F-foundations/brand-identity.md)
  • Write 5-10 voice rules in F-foundations/brand-voice.md
  • Send 3-5 test messages to verify the voice feels right
  • Adjust voice rules based on the output you see

Day 2-3 — Knowledge

  • Add domain knowledge to B-brain/ (industry data, methodologies, client info)
  • Test that agents reference this knowledge in responses
  • Use the ingestion command to import existing documents

Day 4-5 — Workflows

  • Create your first YAML skill (content pipeline is a great start)
  • Test the writer → reviewer pipeline with a real task
  • Iterate on agent prompts based on output quality

Week 2+ — Expansion

  • Connect Google Workspace tools (email, calendar, drive)
  • Add more agents for specialized roles
  • Build more skill workflows for recurring tasks
  • Let the self-evolution engine suggest improvements

7. Test Everything Works

Run through these tests to confirm your system is properly configured.

Test 1: Brand Recognition

Ask your system:

What do you know about my business?

Expected: The response should reference your brand identity, business name, audience, and values — not generic AI filler.

Test 2: Voice Calibration

Ask your system:

Write a social media post about why AI agents should work as a team, not as disconnected tools.

Expected: The output should match your voice rules. If you said "no exclamation marks," there should be none. If you said "short paragraphs," they should be short.

Test 3: System Health (Full edition only)

Open this URL in your browser:

http://localhost:8080/status

Expected: A JSON response showing "healthy" status, connected LLM providers, and active systems.

Test 4: Agent Routing (Full edition only)

curl -X POST http://localhost:8080/api/chat \
  -H "Content-Type: application/json" \
  -d '{"message": "Analyze the competitive landscape for [your industry]", "system_key": "consulting", "user_id": "test-user"}'

Expected: The response should be handled by the Analyst agent (check the agent field in the response), using a more powerful model for complex analysis.

8. Troubleshooting

Common issues and their fixes. If your problem isn't here, ask in the community or email info@realizeos.ai.

Installation Issues

"git is not recognized" or "git: command not found"

Git isn't installed. Download from git-scm.com. After installing, close and reopen your terminal.

"python: command not found" or wrong Python version

Try python3 --version instead. If Python isn't installed, download from python.org. During installation, check "Add Python to PATH." This is the most commonly missed step.

"pip: command not found"

Use python -m pip install -r requirements.txt instead. If that doesn't work, reinstall Python with the "pip" option checked.

Docker Desktop won't start

Windows: Make sure WSL2 is installed. Open PowerShell as admin and run wsl --install. Restart your computer.
Mac: Make sure you downloaded the right version (Apple Silicon vs Intel).
Both: Check that virtualization is enabled in your BIOS settings.

"Port 8080 already in use"

Another process is using that port. Either stop it, or change the port in your .env file to something like API_PORT=8081.

API Key Issues

"Invalid API key" or "Authentication failed"

Check your .env file. Common mistakes:

  • Spaces around the = sign (wrong: ANTHROPIC_API_KEY = sk-...)
  • Quotes around the value (wrong: ANTHROPIC_API_KEY="sk-...")
  • Trailing newline or invisible characters — re-type the key
  • Key was revoked or expired — create a new one at console.anthropic.com

"Insufficient credits" or "Payment required"

Your Anthropic account needs credit. Go to console.anthropic.com → Billing → Add at least $5.

"Rate limit exceeded"

You're sending too many requests. Wait a minute and try again. For production use, increase your rate limits in the Anthropic console or adjust RATE_LIMIT_PER_MINUTE in your .env.

Runtime Issues

"Claude doesn't know my brand voice"

Your F-foundations/brand-voice.md is probably too vague. Vague instructions like "be professional" don't help. Instead, write specific rules: "Use short paragraphs (2-3 sentences). Start with data, not opinions. No exclamation marks. Never use the word 'synergy.'"

"Agents respond generically"

The knowledge base might not be indexed. Run the index command:

python cli.py index

Also check that your agent definitions in A-agents/ have specific, detailed role descriptions — not just "handles content."

"Skill doesn't trigger"

Check the YAML syntax in your skill file. Common issues: incorrect indentation, missing triggers array, or the trigger keywords don't match your message. Test with an exact trigger phrase first.

"KB search returns nothing"

Run the index command to rebuild: python cli.py index. Also check that your knowledge files use UTF-8 encoding and have actual content (not just headers).

Obsidian / Lite Issues

"Obsidian shows empty vault"

You likely opened the wrong folder. Close the vault, then re-open by selecting the folder that contains the FABRIC subfolders (A-agents/, B-brain/, etc.).

"Claude Code doesn't read CLAUDE.md"

Make sure you're running Claude Code from the correct directory — the one containing the CLAUDE.md file. Check with ls or dir that the file exists in your current working directory.

"Can't see files or folders"

In Obsidian, go to Settings → Files & Links → Enable "Detect all file extensions." Some markdown files might be hidden by default filters.
