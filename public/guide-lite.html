How It Works

You speak naturally → Claude reads CLAUDE.md → Picks the right agent → Loads your brand + knowledge → Responds in your voice

Your FABRIC Directory

  • F Foundations — Brand identity, voice rules, core standards
  • A Agents — AI team definitions and routing
  • B Brain — Domain knowledge, market data, expertise
  • R Routines — Skills, workflows, state maps
  • I Insights — Learning log, feedback, decisions
  • C Creations — Output: deliverables, drafts, final assets

Quick Health Check

Open your terminal in the vault directory and start Claude Code:

claude

Then ask:

What do you know about my business?

Expected: A response that references your brand identity, business name, audience, and values — not generic AI filler.

How Routing Works

Claude reads the routing config in realize-os.yaml and picks the right agent based on your message:

  • Content tasks (write, draft, post) → Writer → Reviewer pipeline
  • Strategy tasks (plan, prioritize, advise) → Analyst → Orchestrator
  • Research tasks (analyze, compare, evaluate) → Analyst
  • General tasks (help, think) → Orchestrator

Customizing Agents

Each agent is a .md file in A-agents/. Open any file and edit the role description, tone, constraints, and examples.

Adding New Agents

  1. Create a new .md file in A-agents/ (e.g., client-comms.md)
  2. Define the role, tone, and constraints
  3. Add routing keywords in realize-os.yaml under agent_routing
  4. Auto-discovered on next conversation start

The Review Pipeline

When review_pipeline: true is set, any content task goes through a two-step process: the Writer creates a draft, then the Reviewer evaluates it against your brand standards. Pass = delivered. Fail = Writer iterates with feedback. This ensures consistent quality without manual review.

Working with Skills

Skills are YAML-defined workflows that automate multi-step processes. Instead of giving instructions step by step, you trigger a skill with a natural phrase and it runs the full pipeline.

How to Trigger

Just say the trigger phrase naturally:

Write a blog post about AI operations for consulting firms.

This triggers the Content Pipeline skill: Writer creates draft → Reviewer evaluates against brand voice → pass = delivered, fail = iterate.

Create Your First Custom Skill

Create a new file at R-routines/skills/meeting-prep.yaml:

name: meeting_prep
triggers:
  - "prepare for meeting"
  - "meeting prep"
  - "get ready for meeting"
task_type: strategy
pipeline:
  - analyst
  - writer
  - orchestrator

This skill has the Analyst research context, Writer draft talking points, and Orchestrator create an action plan. Skills are auto-discovered — no restart needed.

Tips for Great Skills

  • Use 3-5 specific trigger phrases per skill
  • Avoid generic triggers that overlap with other skills
  • YAML is sensitive to spacing — use spaces, not tabs
  • Test with an exact trigger phrase first, then try variations

Your First Week Roadmap

Day 1 — Foundations ~30 min

  • Send 5-10 test messages to check your brand voice
  • Refine rules in F-foundations/brand-voice.md based on what you see
  • Add 3-5 anti-patterns (what your AI should NEVER say or do)
  • Complete your F-foundations/brand-identity.md with specific details

Why Anti-Patterns Matter

Anti-patterns have outsized impact. "Never use exclamation marks" or "Never start with 'In today's fast-paced world'" dramatically improves output quality. Add at least 3.

Day 2-3 — Knowledge ~1 hour

  • Add domain knowledge to B-brain/ — industry data, methodologies, past work
  • Test that agents reference this knowledge in responses
  • Drop in any existing documents you want the AI to know about

Day 4-5 — Workflows ~1 hour

  • Create your first custom YAML skill (content pipeline is a great start)
  • Test it with a real task from your actual workflow
  • Iterate on agent prompts — small changes make big differences

Week 2+ — Expansion

  • Add specialized agents for roles you need
  • Build skill workflows for recurring tasks
  • Activate a second venture if you manage multiple businesses
  • Consider upgrading to Full edition for API access and integrations

Refine Brand Voice

Edit F-foundations/brand-voice.md with specific rules:

## Voice Rules
- Use short paragraphs (2-3 sentences max)
- Start with data or a specific observation, not a generic claim
- No exclamation marks
- Never use "synergy", "leverage", or "game-changer"
- Write at a 10th-grade reading level

Feature Flags

Edit realize-os.yaml to toggle features:

features:
  review_pipeline: true    # Auto-route content through Reviewer
  auto_memory: true        # Log learnings to I-insights/
  proactive_mode: true     # Suggest next steps, ask clarifying questions
  cross_system: false      # Share context across ventures

Troubleshooting

Common issues and their fixes. If your problem isn't here, ask in the community or email info@realizeos.ai.

Claude Code doesn't read CLAUDE.md

Make sure you're running claude from the correct directory:

cd /path/to/your/realize-lite-vault
ls CLAUDE.md    # Should show the file
claude          # Now start Claude Code

Output sounds generic — not my voice

Your F-foundations/brand-voice.md is too vague. Replace generic rules like "be professional" with specific ones:

  • "Use short paragraphs (2-3 sentences)"
  • "Start with data or a specific observation"
  • "No exclamation marks"
  • "Never use the word 'synergy'"
  • Paste a real writing sample as a "Good Example"

Skill doesn't trigger

Check YAML syntax (indentation matters — use spaces, not tabs). Verify the triggers array contains phrases that match your message. Test with an exact trigger phrase first.

Obsidian shows empty vault

You opened the wrong folder. Close the vault, then re-open by selecting the folder that contains the FABRIC subfolders (A-agents/, B-brain/, etc.). If you unzipped into a nested folder, look one level deeper.

Agents respond generically — don't reference my knowledge

Check that A-agents/ contains detailed agent definitions with specific role descriptions. Add more context to B-brain/. Make sure F-foundations/brand-identity.md has specific business details.

"claude: command not found"

Node.js isn't installed or isn't in your PATH. Close and reopen your terminal after installing Node.js. On macOS: brew install node. On Windows: download from nodejs.org and check "Add to PATH" during install.
