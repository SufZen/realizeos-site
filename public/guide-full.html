Connect Your AI Tools (MCP Toolkit)

⚡ The Fastest Way to Talk to Your Engine

Docker Desktop includes a built-in MCP Toolkit that lets you connect Claude Code Desktop, Claude Code CLI, and Gemini CLI to your running RealizeOS container — with just a few clicks. No manual API calls needed.

What Is MCP?

The Model Context Protocol (MCP) is an open standard that lets AI tools (like Claude Code and Gemini) communicate with external services. Docker Desktop’s MCP Toolkit turns your running containers into MCP servers automatically, so your AI tools can call your RealizeOS engine directly.

Prerequisites

  • Docker Desktop 4.62+ — MCP Toolkit is built-in from this version
  • Your RealizeOS container running (via docker compose up -d)
  • At least one AI tool installed: Claude Code Desktop, Claude Code CLI, or Gemini CLI

Step-by-Step Setup

  1. Open Docker Desktop and verify your RealizeOS container is running (green status).
  2. Navigate to the MCP Toolkit — find it in the Docker Desktop sidebar (look for the MCP Toolkit or Extensions section).
  3. Go to the “Clients” tab in the MCP Toolkit. You’ll see a list of supported AI tools.
  4. Enable your preferred clients:
    • Claude Code Desktop — toggle on, Docker configures it automatically
    • Claude Code CLI — toggle on for terminal-based usage
    • Gemini CLI — toggle on if you use Google’s Gemini tools
  5. Restart your AI tool if it was already open. The MCP connection is detected on startup.

Verify the Connection

After enabling, verify your AI tool can see the Docker MCP gateway:

# Claude Code CLI — list MCP servers
claude mcp list

# You should see the Docker MCP gateway listed

For Claude Code Desktop, open the app and check Settings → MCP Servers. The Docker gateway should appear automatically.

How It Works Behind the Scenes

The MCP Toolkit registers a gateway that proxies MCP requests to your container:

# The gateway config that Docker Desktop registers:
{
  "command": "docker",
  "args": ["mcp", "gateway", "run"]
}

This runs docker mcp gateway run which connects your AI tool to all MCP-enabled containers, including RealizeOS.

What You Can Do After Connecting

Once connected, your AI tool can interact with RealizeOS directly. Instead of writing curl commands, you just talk to Claude or Gemini naturally:

  • “Ask my RealizeOS consultant agent to draft a proposal”
  • “Check the health status of my RealizeOS engine”
  • “List all agents in my consulting system”
  • “Run a skill on my RealizeOS instance”

✓ Why This Matters

The MCP Toolkit is the recommended way to interact with your RealizeOS engine day-to-day. It eliminates the need to memorize API endpoints or write curl commands — your AI tools become a natural interface to your entire operations system.

⚠ Troubleshooting

  • Don’t see MCP Toolkit? Update Docker Desktop to version 4.62 or later.
  • Gateway not appearing in Claude Code? Restart the AI tool after enabling in Docker Desktop.
  • Container not detected? Make sure RealizeOS is running (docker compose ps).
  • Manual config needed? You can also add the gateway manually to your AI tool’s MCP config with: {"command": "docker", "args": ["mcp", "gateway", "run"]}

The system auto-routes to the right agent based on your message, but you can also be explicit:

  • “Ask the analyst to research competitors in the AI consulting space”
  • “Have the writer draft a LinkedIn post about our latest project”
  • “Ask the reviewer to check this proposal for brand consistency”

What Happens Behind the Scenes

When you send a message, the engine:

  1. Identifies the right agent (or uses the one you specified)
  2. Routes to the optimal LLM (Flash for simple, Sonnet for content, Opus for complex)
  3. Builds a 12-layer prompt from your knowledge base (brand voice, templates, context)
  4. Executes any needed tools (search, calendar, email)
  5. Returns the response through your AI tool

🔧 For developers: direct API access

You can also interact with the engine via REST API. See the Customizing & Growing step for the full API reference and curl examples.

  - "write a post"
  - "create content"
  - "draft an article"

  - writer

v2: Multi-Step with Tools

name: competitor_analysis
version: "2.0"
description: "Research competitors and produce analysis"
triggers:
  - "analyze competitor"
  - "competitive analysis"
task_type: research
steps:
  - id: search
    type: tool
    action: web_search
    label: "Search for competitor info"
    params:
      query: "{user_message} site:linkedin.com OR site:crunchbase.com"

  - id: analyze
    type: agent
    agent: analyst
    label: "Analyze findings"
    inject_context: [search]

  - id: confirm
    type: human
    question: "Should I create a detailed report?"

  - id: report
    type: agent
    agent: writer
    label: "Write the report"
    inject_context: [search, analyze]

Connecting Tools & Integrations

8a

Telegram Bot

  1. Open Telegram and message @BotFather
  2. Send /newbot and follow the prompts
  3. Copy the bot token and add to your .env:
TELEGRAM_BOT_TOKEN=your_token_here

Start the bot:

python cli.py bot

Or add to Docker and restart: docker compose up -d. Now message your bot on Telegram — it routes through the same agent system.

8b

Google Workspace (Gmail, Calendar, Drive)

  1. Go to Google Cloud Console → Create/select project
  2. Enable Gmail API, Google Calendar API, Google Drive API
  3. Create OAuth 2.0 credentials (Desktop app) → download JSON
  4. Save as .credentials/credentials.json
  5. Run authorization:
python cli.py auth google

This opens a browser for OAuth consent. After authorizing, tokens are saved automatically.

13 Google Workspace Tools

Gmailgmail_search, gmail_read, gmail_send, gmail_create_draft
Calendarcalendar_list_events, calendar_create_event, calendar_update_event, calendar_find_free_time
Drivedrive_search, drive_list_folder, drive_read_content, drive_create_doc, drive_append_doc

Write operations (send, create, update) always require confirmation before executing.

8c

Web Search (Brave API)

  1. Get a Brave Search API key
  2. Add to .env:
BRAVE_API_KEY=your_key_here

Restart the server. Web search is available in skills and direct queries.

8d

Make.com / n8n / Zapier

Connect any automation platform via the REST API:

  1. Create a new scenario/workflow
  2. Add an HTTP module (POST request)
  3. Configure:
URL:     http://YOUR_SERVER_IP:8080/api/chat
Method:  POST
Headers: Content-Type: application/json
         Authorization: Bearer YOUR_API_KEY
Body:
{
  "message": "{{trigger_data}}",
  "system_key": "consulting",
  "user_id": "make-automation"
}

Example flow: New email in Gmail → summarize with RealizeOS → post summary to Slack.

8e

MCP Servers

MCP (Model Context Protocol) lets you connect external tool servers. The system auto-discovers and registers all tools exposed by MCP servers, making them available to agents and skills.

8f

Inbound Webhooks

External services (GitHub, Stripe, forms) can trigger your agents by posting to the /api/chat endpoint. The message is processed like any other request — routed to the right agent with full context.

Works with GitHub webhooks, Stripe events, form submissions, or any HTTP-capable system. See the API reference for endpoint details.

Self-Evolution

The Full edition includes a self-evolution engine that continuously improves your system based on usage patterns.

How It Works

  1. Interaction Tracking — Every request logged with metadata (agent, task type, duration)
  2. Gap Detection — Finds unhandled patterns and repeated ad-hoc requests
  3. Skill Suggestion — Auto-generates skill YAML for detected gaps
  4. Prompt Refinement — Suggests agent prompt improvements from feedback

Example

After you manually create competitor analyses 5 times, the system detects the pattern:

Gap detected: "competitor analysis" requested 5 times with no matching skill.
Suggested skill: competitor_analysis.yaml
Approve? [yes/no]

All changes require your explicit approval. Nothing is modified automatically.

Risk-Gated Approval

  • Low risk (new skill, prompt tweak) → Simple yes/no
  • Medium risk (agent modification) → Shows diff before applying
  • High risk (config change) → Requires explicit confirmation with details

Troubleshooting

Common issues and their fixes. If your problem isn't here, ask in the community or email info@realizeos.ai.

Docker Issues

"Docker daemon not running"

Make sure Docker Desktop is running (check system tray).

Windows: Ensure WSL2 is installed:

wsl --install

Restart your computer after installing WSL2.

"Port 8080 already in use"

Change the port in .env: API_PORT=8081. Then update your docker-compose or CLI command.

API Issues

"401 Unauthorized"

If REALIZE_API_KEY is set in .env, you must include Authorization: Bearer YOUR_KEY in every request.

"System not found"

Your system_key doesn't match any entry in realize-os.yaml. Check with:

curl http://localhost:8080/api/systems

LLM Issues

"Invalid API key" or "Authentication failed"

Check your .env file: no spaces around =, no quotes, no trailing spaces. Verify the key hasn't been revoked.

"Insufficient credits"

Add credit to your API provider. Anthropic: console.anthropic.com → Billing → Add at least $5.

Config Issues

"Config reload failed"

YAML syntax error. Validate:

python -c "import yaml; yaml.safe_load(open('realize-os.yaml'))"

"Output sounds generic"

Your F-foundations/brand-voice.md needs more specific rules. Add concrete do/don't rules and paste real writing samples.

"Wrong agent handles my task"

Add more keywords to agent_routing in realize-os.yaml, or force an agent with agent_key in the API call.
